## 1. 一屏总览（先看这里）

```text
前端展示层：src/views + src/components + src/layout
前端状态与路由：src/store + src/router + src/api
```

## 3. 目录职责（精简版）

| 目录 | 职责 | 改动风险 |
|---|---|---|
| `src/views` | 页面业务实现（管理端 + 客户端） | 中：最易影响交互流程 |
| `src/components` | 通用组件与业务组合件 | 中：会波及多页复用 |
| `src/router` | 路由、守卫、预加载策略 | 高：易引发白屏/越权/跳转循环 |
| `src/store` | 登录态、权限态、客户端会话与缓存 | 高：易导致跨账号数据残留 |
| `src/api` | 前后端接口契约封装 | 中：契约错配会引发全局报错 |
| `backend/src/routes` | API 路由与参数入口 | 高：直接影响接口行为与权限 |
| `backend/src/middleware` | 鉴权、CSRF、错误处理 | 高：全局安全与稳定性核心 |
| `backend/src/services` | 业务事务、并发控制、审计副作用 | 高：改动会影响数据一致性 |
| `backend/src/entities` | ORM 实体映射 | 高：关系到迁移与查询正确性 |
| `backend/src/config` | 环境装载、数据源、SQLite 启动策略 | 高：会影响启动与部署 |
| `backend/src/utils` | 安全、令牌、日志脱敏、工具函数 | 中-高：复用范围大 |
| `backend/sql` | 结构变更 SQL 与迁移脚本 | 高：涉及历史数据与回滚 |
| `scripts` / `backend/scripts` | 构建、回归、并发验收脚本 | 中：影响 CI 结果可信度 |
| `docker` + `compose*.yml` | 容器化部署与网关配置 | 高：影响线上可用性与暴露面 |

`src/views/system/DatabaseMigrationView.vue` → `backend/src/routes/data-maintenance.routes.ts` → `backend/src/services/database-migration.service.ts` → `backend/src/config/*` + `backend/sql/*`

## 5. 改动前检查清单（实用）

1. 这是页面问题、接口问题，还是数据一致性问题？
2. 是否涉及鉴权/CSRF/审计（涉及则必看 `middleware + audit.service`）？
3. 是否涉及并发写（涉及则必看事务与重试逻辑）？
4. 是否涉及 SQLite 与 MySQL 双方言差异？
5. 是否需要同步更新验证脚本与文档？

# 登录页极简几何重构计划

## 摘要
当前登录页（LoginView.vue）在之前的优化中偏向了“面向开发者的技术控制台”风格，存在文字说明过多、线框密集等问题。本计划将严格按照用户的指示，采用真正的“包豪斯（Bauhaus） / Apple 现代几何极简”设计语言对其进行重构。即：去除多余的系统指令文字、取消工程网格底纹、改用纯粹的几何形体（大圆角、实体色块、高对比度留白）与原生的交互反馈。

## 当前状态分析
目前的 `src/views/auth/LoginView.vue` 包含了大量的控制台说明文字（如“AUTH NODE 01”、“极简认证控制台”等说明性占位文本）和细线框背景（`linear-gradient` 绘制的网格），这违背了面向最终业务用户的极简审美，属于“AI 认知错位”带来的重工业后台感。

## 提议的变更
1. **替换目标文件**：`F:\Y-Link\src\views\auth\LoginView.vue`
2. **重构内容**：
   - **布局**：采用经典的左右分栏（Split-Screen）版式。
   - **左侧视觉区**：纯粹的几何艺术表达（正圆、胶囊体、中空圆环），去除所有多余解释性文字，建立高级品牌感。
   - **右侧操作区**：极致干净的表单区，仅保留输入框、交互型几何按钮（`Right` 图标 hover 动效）、以及极轻量级的测试账号提示。
   - **样式细节**：
     - 移除 `blur` 和弥散光发光效果。
     - 引入原生的 `52px` 坚实输入框与 `56px` 按钮高度，使用 `16px` 大圆角。
     - 配置完整的亮暗色 CSS 变量（`--bg-primary`, `--bg-panel`, `--accent` 等），实现丝滑的亮暗主题切换。
3. **保留底层逻辑**：
   - 现有的真实 `authStore.login` 链路。
   - `resolveSafeRedirect` 安全回跳机制。
   - `extractErrorMessage` 错误提示。
   - Element Plus 的表单验证（`rules`）。

## 验证步骤
1. 将用户提供的极简重构代码写入 `F:\Y-Link\src\views\auth\LoginView.vue`。
2. 运行 `GetDiagnostics` 检查语法及类型错误。
3. 执行前端构建验证 (`npm run build`)，确保重构没有破坏编译链路。

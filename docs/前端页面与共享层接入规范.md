# Y-Link 前端页面与共享层接入规范

> 文档更新时间：2026-05-01
> 适用对象：前端开发、联调开发、维护接手人员

## 1. 文档目的
- 本文档用于统一管理端与客户端的前端接入方式，避免继续在页面里散落原生弹层、散点权限判断、重复请求治理和错误提示逻辑。
- 本文档是 `Task 3 / Task 5` 的执行规范，新增页面、重构旧页面、修复高频问题时必须优先遵守本文，而不是继续复制历史页面写法。

## 2. 页面分层约定
- 页面入口负责“装配”：
  - 负责路由参数、页面标题、业务区块排列、页面级状态组合。
  - 不负责沉淀重复弹层、重复列表壳、重复权限门禁工具。
- 共享层负责“通用能力”：
  - `src/components/common`：共享页面壳、弹窗壳、抽屉壳、分页条、工具栏卡片。
  - `src/composables`：请求调度、权限动作、设备判断、路由绑定等跨页面能力。
  - `src/utils`：错误归一化、缓存归一化、打印/PDF 之类稳定工具。
- 业务页负责“本页独有规则”：
  - 只保留当前业务域独有的字段映射、提交校验、视图文案与联动逻辑。

## 3. 路由与菜单规范
- 所有业务页路由必须使用懒加载 `() => import()`。
- 路由 `meta` 必须显式声明页面语义：
  - `title`：页面标题。
  - `menu`：是否进入导航。
  - `requiredPermissions` 或 `requiredAnyPermissions`：入口可见与访问权限来源。
  - `allowedRoles`：仅在明确存在角色硬边界时使用，例如供货方独立入口。
  - `keepAlive` 与 `preloadTargets`：高频路径按性能治理规范配置。
- 菜单可见性、路由守卫、页面按钮显隐必须共享同一套权限来源，禁止页面自己再维护额外白名单。

## 4. 权限动作规范
- 页面按钮显隐使用 `computed(() => hasPermission('xxx'))`。
- 点击前兜底使用 `usePermissionAction`：
  - `hasPermission`：只做显隐。
  - `ensurePermission`：做点击前门禁与统一提示。
  - `runWithPermission`：适合一段异步动作整体包裹。
- 禁止新增页面级 `showPermissionDenied()` 直呼散点逻辑。
- 禁止把“角色判断 + 权限判断”写死在页面内，除非该页面本身就是角色专属入口。

## 5. 弹层与抽屉规范
- 短内容弹窗优先使用 `BizCrudDialogShell`。
- 长内容弹窗优先使用 `BizCrudDialogShell`，并根据内容复杂度选择：
  - `height-mode="auto"`：短表单、确认类弹窗。
  - `height-mode="scroll"`：长表单、工作台式编辑区。
- 详情抽屉、移动端编辑抽屉优先使用 `BizResponsiveDrawerShell`。
- 已接入共享壳后，页面内部禁止再叠加新的主滚动容器，例如：
  - 不要再写 `max-h-[55vh] overflow-auto` 作为主列表滚动区。
  - 若必须局部滚动，需明确是“次级局部滚动”，不能抢占主滚动职责。
- 原生 `el-dialog / el-drawer` 仅允许出现在以下场景：
  - 共享壳自身实现文件。
  - 经评审确认的专用壳组件内部，例如扫码专用弹窗。

## 6. 列表与请求规范
- 列表页统一使用分页状态工具与共享列表壳：
  - `createPaginatedListState`
  - `applyPaginatedResult`
  - `BizResponsiveDataCollectionShell`
  - `PagePaginationBar`
- 高频列表、筛选、搜索、详情拉取必须走 `useStableRequest`，防止旧请求回写覆盖新结果。
- 页面中不要直接把请求结果拆散写入多个 ref；优先维护统一 `listState / detailState / formState`。

## 7. 错误提示规范
- 接口错误优先使用 `extractErrorMessage(error, '默认提示')`。
- 列表加载失败、提交失败、导出失败等场景必须给用户稳定中文提示。
- 禁止直接把原始异常对象或调试用英文文案暴露到界面。

## 8. 页面样式规范
- 样式优先落在共享壳和通用类，不要在业务页堆叠大量全局例外选择器。
- 若页面问题属于“弹窗高度、抽屉滚动、overlay 对齐”这类结构性问题，应优先修共享壳或 `src/style.css` 的共享安全网。
- 页面 `scoped` 样式只处理本页独有视觉，不承担全局挂载到 `body` 的弹层修补职责。

## 9. 推荐接入模板
- 列表页：
  - 页面入口装配 `PageContainer + PageToolbarCard + BizResponsiveDataCollectionShell + PagePaginationBar`
  - 请求走 `useStableRequest`
  - 权限动作走 `usePermissionAction`
- 编辑页：
  - 短表单弹窗走 `BizCrudDialogShell height-mode="auto"`
  - 长表单编辑走 `BizCrudDialogShell height-mode="scroll"`
- 移动端明细编辑：
  - 走 `BizResponsiveDrawerShell`
  - 主滚动由壳层承接

## 10. 回归检查清单
- 是否仍新增了原生 `el-dialog / el-drawer` 散点实现。
- 是否在页面里新增了 `showPermissionDenied()` 散点调用。
- 是否高频请求未接入 `useStableRequest`。
- 是否按钮显隐、点击门禁、路由访问存在三层权限分裂。
- 是否共享壳已接入，但页面内部又新增主滚动容器造成双滚动。

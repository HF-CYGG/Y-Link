# Y-Link 模块职责地图

> 文档更新时间：2026-05-01
> 适用对象：二开接手、联调开发、权限改造、回归测试

## 1. 文档说明
- 本文档用于回答两个核心问题：
  - 某个业务域对应哪些前端页面、API、后端路由、服务与权限点。
  - 出问题时应该先从哪里定位，而不是全仓库到处猜。

- 前端 API：
  - `src/api/modules/auth.ts`

  - `backend/src/routes/auth.routes.ts`

## 10. 排障建议
- 页面看不到入口：先查 `routes.ts` 的权限与角色元信息。
- 页面能看到但点不开：先查 `usePermissionAction` 或页面按钮显隐条件。
- 接口 403：先查后端 routes 的 `requirePermission / requireRole`。
- 状态写入异常：先查 service 是否有事务与审计。
- 弹层高度/滚动异常：先查共享壳，再查页面是否又加了局部滚动容器。

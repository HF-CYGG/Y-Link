# Y-Link 系统配置、通知中心与数据库迁移

## 适用范围

- 适用于系统配置键、订单流水号、O2O 规则、验证码通道、通知中心、SQLite/MySQL 迁移和数据维护。
- 适用于定位“这个配置实际存在哪、通知为什么没发、迁移为什么不能切库”的问题。

## 最后核对来源

- `backend/src/routes/system-config.routes.ts`
- `backend/src/routes/notification.routes.ts`
- `backend/src/routes/data-maintenance.routes.ts`
- `backend/src/services/system-config.service.ts`
- `backend/src/services/notification.service.ts`
- `backend/src/services/database-migration.service.ts`
- `backend/src/services/data-maintenance.service.ts`

## 真实入口

- 系统配置入口：`system-config.routes.ts`
- 通知中心入口：`notification.routes.ts`
- 数据维护与迁移入口：`data-maintenance.routes.ts`
- 真规则源：`system-config.service.ts`、`notification.service.ts`、`database-migration.service.ts`

## 前端链路

- 系统配置页按配置分区展示：流水号、O2O 规则、客服配置、验证码通道、部门树、职工目录。
- 通知中心页管理规则、在线窗口、测试发送和站内收件箱。
- 数据库迁移助手页承接预检、任务创建、执行、切换、回滚和运行时覆盖状态查看。

## 后端链路

- `system-config.service.ts` 维护多组 `configKey/configGroup`，重点包括：
  - 订单流水号起始值/当前值/宽度
  - O2O 自动取消、限购、门店展示文案
  - 客服在线/离线文案与 FAQ
  - 验证码移动端/邮箱通道配置
  - 客户端部门树和职工目录
- `notification.service.ts` 负责：
  - 默认规则补齐
  - 在线窗口配置
  - 规则更新
  - 在线态快照
  - 站内收件箱
  - 邮件与飞书外发
  - 测试发送
- `database-migration.service.ts` 负责：
  - SQLite -> MySQL 预检
  - 迁移任务落盘
  - SQLite 物理备份 + JSON 快照双备份
  - 表迁移、校验、运行时覆盖写入、回滚
- `data-maintenance.service.ts` 负责 SQLite 备份与 JSON 导入导出。

## 关键状态/字段/快照

- 流水号配置不仅有 `start/current/width`，还会做在用单号安全检查。
- 客服在线判断依赖工作时间和在线窗口秒数，而不是单纯看用户是否存在。
- 通知规则包含：站内接收人、邮件接收人、飞书 Webhook、外发时机、被监控账号列表。
- 迁移任务会落盘记录状态、进度、备份文件、校验结果和运行时覆盖是否已写入。

## 权限与安全边界

- 配置读取通常是 `system_configs:view`，配置修改通常是 `system_configs:update`，高风险修改还要求 `admin`。
- 通知规则测试发送、数据库迁移执行、运行时覆盖切换都属于高风险治理操作。
- 飞书 Webhook、验证码通道 URL 等外部地址必须经过安全校验，不能指向危险主机。

## 常见异常与排查顺序

1. 某配置改了没生效：查实际 `configKey` 是否正确，服务层是否做了额外安全校验或缓存重建。
2. 流水号改完报冲突：查系统配置服务的占用快照与预订单/出库单在用编号。
3. 通知没外发：先查规则是否命中，再查 `shouldTriggerExternal()`、接收人解析和 dispatch 记录。
4. 飞书或邮件测试发送失败：查 URL/模板/成功匹配串是否合规。
5. 迁移任务跑不动：先看 precheck，再看任务状态、校验结果和运行时覆盖状态。

## 验证与回归关注点

- 配置改动回归：读取、保存、回显、权限。
- 通知改动回归：规则列表、在线快照、测试发送、站内收件箱未读数。
- 迁移改动回归：预检、创建任务、执行任务、查看运行时覆盖、回滚入口。

# Y-Link 路由、壳层与预热机制

## 适用范围

- 适用于改动管理端/客户端路由、菜单、快捷入口、预热策略、KeepAlive 和共享壳层时。
- 适用于判断某页面为什么会被缓存、为什么会预热、为什么菜单高亮不对。

## 最后核对来源

- `src/router/routes.ts`
- `src/router/route-performance.ts`
- `src/layout/AppLayout.vue`
- `src/views/client/components/ClientMainLayout.vue`
- `src/router/index.ts`

## 真实入口

- 管理端与客户端路由真源：`src/router/routes.ts`
- 懒加载与预热策略真源：`src/router/route-performance.ts`
- 管理端主壳层：`src/layout/AppLayout.vue`
- 客户端主壳层：`src/views/client/components/ClientMainLayout.vue`

## 前端链路

- 所有业务页先在 `routeViewLoaders` 中注册异步加载器，再在 `routes.ts` 里挂入路由树。
- 管理端菜单、工作台快捷入口和客户端底部导航都不是额外常量，而是从路由 `meta` 派生。
- 管理端的关键 `meta` 字段包括：`title`、`menu`、`menuOrder`、`menuGroup`、`shortcut`、`requiredPermissions`、`requiredAnyPermissions`、`allowedRoles`、`keepAlive`、`preloadTargets`、`viewKey`、`suppressGlobalLoadingBar`。
- 客户端的关键 `meta` 字段包括：`requiresClientAuth`、`clientGuestOnly`、`clientNav`、`clientNavOrder`、`hideClientBottomNav`、`preloadTargets`。
- `AppLayout.vue` 通过 `router-view + Suspense + KeepAlive` 组合承接管理端页面，并用 `resolveViewKey()` 保证共享壳层路由的缓存键稳定。

## 后端链路

- 路由本身不直接决定后端行为，但它的权限 `meta` 必须和 `backend/src/routes/*` 的 `requirePermission` / `requireRole` 保持一致。
- 若菜单可见但接口 403，通常不是前端组件问题，而是前后端权限元数据失配。

## 关键状态/字段/快照

- `viewKey` 用于把多个入口映射到同一共享壳层实例，例如产品中心、用户中心、供应方工作台。
- `keepAlive` 决定页面是否复用组件实例；高频查询页通常开启，低频治理页不一定开启。
- `preloadTargets` 不是随手补全，而是“高概率下一跳”列表。
- `deferPreloadOnColdStart` 用于避免系统管理等重页面在冷启动时过度抢占带宽。
- `suppressGlobalLoadingBar` 用于共享壳层内部标签切换时避免制造整页刷新的错觉。

## 权限与安全边界

- 管理端路由守卫依据 `requiresAuth + requiredPermissions/requiredAnyPermissions + allowedRoles`。
- 客户端路由守卫依据 `requiresClientAuth` 与 `clientGuestOnly`。
- 供应方入口是少数同时强依赖角色和权限边界的分支，修改时必须核对 `allowedRoles: ['supplier']`。

## 常见异常与排查顺序

1. 菜单不显示：先查 `routes.ts` 中 `menu/menuOrder/menuGroup`，再查当前用户权限集。
2. 菜单高亮错位：查 `activeMenu`、父子路由 path、共享壳层 `viewKey`。
3. 页面切换时旧内容残留：查 `keepAlive`、`resolveViewKey()`、共享壳层是否复用同一组件。
4. 冷启动变慢：查 `route-performance.ts` 中该页面和相邻页面的 `preloadTargets`。
5. 客户端底部导航缺失：查 `clientNav`、`clientNavOrder`、`hideClientBottomNav`。

## 验证与回归关注点

- 新增路由后至少检查：菜单是否出现、权限是否匹配、首跳是否正确、移动端/桌面端布局是否正常。
- 调整预热后关注：`npm run build` 是否通过，冷启动是否拉起过多页面子包。
- 改共享壳层时重点回归：产品中心、用户中心、供应方工作台、系统管理页。

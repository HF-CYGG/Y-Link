# Y-Link O2O 预订单、核销、退货与正式出库联动

## 适用范围

- 适用于 O2O 下单、改单、核销、退货、超时回收、库存日志和正式出库单联动场景。
- 适用于定位“为什么库存先占用、为什么核销后生成出库单、为什么退货有时回预占有时回现货”的问题。

## 最后核对来源

- `src/api/modules/o2o.ts`
- `src/views/o2o/*`
- `src/views/client/ClientOrdersView.vue`
- `src/views/client/ClientOrderDetailView.vue`

## 真实入口

- 客户端下单入口：`POST /api/o2o/mall/preorders`
- 客户端改单/撤销/退货入口：`/api/o2o/mall/preorders/*`
- 管理端查询/核销/现场改单入口：`/api/o2o/orders*`、`/api/o2o/verify*`
- 服务真源：`o2o-preorder.service.ts`

## 前端链路

- 客户端订单列表和详情页主要围绕 `pending / verified / cancelled` 展示，并附带状态报告对象。
- 管理端有三类入口：订单查询、核销控制台、入库/库存日志协同页。
- O2O 前端既要显示预订单本身，也要展示是否已产生正式出库单、是否存在退货申请、是否有商家留言和业务状态。

## 后端链路

- `o2o-preorder.service.ts` 是完整闭环真源，负责：
  - 商城商品列表和门店配置
  - 预订单创建、查询、详情、摘要
  - 客户端改单、撤销和退货申请
  - 管理端订单查询、业务状态、商家留言、现场改单
  - 预订单核销、退货核销
  - 超时自动取消与库存释放
  - 正式出库单创建
  - 库存日志写入
- `o2o.routes.ts` 明确分成：
  - 客户端匿名商品接口
  - 客户端鉴权后的订单接口
  - 管理端查询/核销/库存接口

## 关键状态/字段/快照

- 核心库存口径：
  - `currentStock`：现货库存
  - `preOrderedStock`：预订单占用库存
- 预订单核心状态：`pending`、`verified`、`cancelled`，并叠加 `businessStatus`、`statusReport`。
- 价格快照采用 `originalPrice`、`discountRate`、`discountedPrice`、`unitPrice`、`lineAmount`，避免实时产品价格污染历史订单。
- 预订单核销成功后会生成正式出库单，并用特定 `idempotencyKey` 前缀与出库单建立联动。
- 退货核销是否回现货库存，取决于原订单在退货时的来源状态。

## 权限与安全边界

- 客户端所有订单动作都必须经过 `requireClientAuth`。
- 管理端查询通常依赖 `orders:view`，核销与正式出库联动依赖 `orders:create` 或 `orders:update`。
- 删除订单池单据还需要 `orders:delete`、管理员角色和永久删除密码。
- 客户端与管理端不允许各自维护不同的价格/库存真相，必须以服务层快照为准。

## 常见异常与排查顺序

1. 下单后库存没占用：查服务层是否写了 `preOrderedStock` 和库存日志。
2. 核销后没生成正式出库单：查 `verifyPreorderInManager()` 和出库单创建逻辑。
3. 退货后库存回错位置：查退货来源订单状态和 `applyReturnVerificationForItem()`。
4. 订单状态展示奇怪：查 `statusReport` 与 `businessStatus` 是否同时生效。
5. 同一订单删了出库单后显示异常：查正式出库单与预订单可见性联动。

## 验证与回归关注点

- 客户端回归：下单、改单、撤销、退货申请、订单详情。
- 管理端回归：订单查询、业务状态、商家留言、现场改单、核销、退货核销。
- 联动回归：库存日志、正式出库单生成、超时回收、删除治理后的可见性。

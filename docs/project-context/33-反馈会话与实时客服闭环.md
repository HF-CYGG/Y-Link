# Y-Link 反馈会话与实时客服闭环

## 适用范围

- 适用于客户端反馈创建、消息追加、会话状态机、客服回复、指派、内部备注和 SSE 实时更新场景。
- 适用于定位“为什么消息没实时推到客服端、为什么状态不能改、为什么负责人变更后有系统消息”的问题。

## 最后核对来源

- `src/views/client/ClientFeedbackView.vue`
- `src/views/client/ClientFeedbackCreateView.vue`
- `src/views/client/ClientFeedbackDetailView.vue`
- `src/views/system/CustomerServiceWorkbenchView.vue`

## 真实入口

- 客户端入口：反馈列表、新建反馈、反馈详情
- 管理端入口：`/system/customer-service`
- 客户端后端路由：`client-feedback.routes.ts`
- 客服后端路由：`customer-service.routes.ts`
- 业务真源：`client-feedback.service.ts`

## 前端链路

- 客户端侧负责创建会话、追加消息、查看客服回复、确认解决、撤回会话和满意度反馈。
- 管理端客服工作台负责：列表筛选、详情、回复、接单/转派、状态更新、结构化问题字段、内部备注。
- 前后端都依赖 SSE 实时通道刷新消息、状态和负责人变化，而不是单纯轮询列表。

## 后端链路

- `client-feedback.service.ts` 是闭环真源，负责：
  - 会话编号生成
  - 客户端消息与客服消息创建
  - 客户端/客服已读状态维护
  - 客户端状态机（待处理、处理中、已解决、已撤回等）
  - 负责人校验与指派
  - 结构化问题字段、内部备注
  - SSE 发布事件
- `customer-service.routes.ts` 明确区分查看、回复、指派、状态变更、字段编辑和内部备注接口。

## 关键状态/字段/快照

- 核心会话字段：`conversationNo`、`status`、`priority`、`assignee*`、`subject`、`issueType`、`tagJson`、`internalRemark`。
- 客服动作往往会创建系统消息，作为“接单/转派/状态变化”的会话内可见痕迹。
- 已读状态不是简单前端本地状态，而是会话级服务端状态。
- SSE 事件不仅推消息，也推负责人变更、字段变更和内部备注更新。

## 权限与安全边界

- 客户端只允许操作自己的会话，服务层会在 `requireOwnedConversation()` 二次校验。
- 客服工作台查看依赖 `customer_service:view`，回复/改状态/指派等依赖 `customer_service:reply`。
- 负责人必须是可分配客服用户，不能把任意后台账号写成负责人。
- 内部备注属于客服侧内部字段，客户端不应可见。

## 常见异常与排查顺序

1. 客服端收不到新消息：查 SSE 通道是否建立，再查 `publishConversationEvent()` 是否触发。
2. 负责人变更后列表没更新：查系统消息发布和 SSE 事件类型。
3. 客户端明明是自己的会话却被拒绝：查 `requireOwnedConversation()` 的 userId 对比。
4. 会话状态改不了：查当前状态机允许的变更路径和角色权限。
5. 内部备注写了但详情没显示：查详情接口是否构建了内部备注视图。

## 验证与回归关注点

- 客户端回归：创建、发消息、查看详情、确认解决、撤回、满意度。
- 客服端回归：列表、详情、回复、接单、转派、改状态、改字段、内部备注。
- 实时回归：客户端发消息后客服端是否实时更新，客服回复后客户端是否实时更新。

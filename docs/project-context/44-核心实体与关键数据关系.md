# Y-Link 核心实体与关键数据关系

## 适用范围

- 适用于理解商品、SKU、出库单、入库单、预订单、退货、通知、会话和审计之间的关系。
- 适用于定位“这个字段是快照还是真实值、这个库存从哪一层聚合、两个单据为什么会联动”的问题。

## 最后核对来源

- `backend/src/config/data-source.ts`
- `backend/src/entities/base-product.entity.ts`
- `backend/src/entities/base-product-sku.entity.ts`
- `backend/src/entities/biz-outbound-order*.entity.ts`
- `backend/src/entities/biz-inbound-order*.entity.ts`
- `backend/src/entities/o2o-preorder*.entity.ts`
- `backend/src/entities/o2o-return-request*.entity.ts`
- `backend/src/entities/inventory-log.entity.ts`
- `backend/src/entities/notification-*.entity.ts`
- `backend/src/entities/client-feedback-*.entity.ts`

## 真实入口

- 实体总清单：`backend/src/config/data-source.ts` 中的 `appEntities`
- 各实体定义位于 `backend/src/entities/*`

## 前端链路

- 前端页面大多消费的是服务层构建后的 View/DTO，而不是直接暴露实体全字段。
- 当前端显示历史订单价格、历史产品名、退货可用数量时，通常依赖的是实体快照字段而不是实时产品表。

## 后端链路

- 商品域：`BaseProduct` <-> `BaseProductSku` <-> `RelProductTag`。
- 出库域：`BizOutboundOrder` <-> `BizOutboundOrderItem`。
- 入库域：`BizInboundOrder` <-> `BizInboundOrderItem`。
- O2O 域：`O2oPreorder` <-> `O2oPreorderItem`，并派生 `O2oReturnRequest` <-> `O2oReturnRequestItem`。
- 库存追踪域：`InventoryLog` 负责记录预占、核销、退货、手工入库等变化。
- 通知域：`NotificationRule`、`NotificationEvent`、`NotificationInbox`、`NotificationDispatch`。
- 客服域：`ClientFeedbackConversation`、`ClientFeedbackMessage`。
- 会话域：管理端 `SysUserSession`，客户端 `ClientUserSession`。

## 关键状态/字段/快照

- 商品与 SKU：
  - 实时库存用 `currentStock`、`preOrderedStock`
  - O2O 价格快照在订单明细里固化，不应回表重算
- 出库单：
  - 历史展示依赖 `productNameSnapshot`、`lineAmount`
- 预订单：
  - 同时持有状态、业务状态、核销码、金额快照、退货统计、是否已删等信息
- 退货申请：
  - 独立状态机，不等于预订单状态本身
- 通知事件和外发记录：
  - 事件落库后，站内通知与外发记录再分开写入

## 权限与安全边界

- 不是所有实体都直接对前端开放；多数都要经过服务层裁剪和脱敏。
- 会话、通知、审计、验证码和运行时覆盖类数据属于高敏感治理对象。
- 删除治理要区分软删除、硬删除、可见性联动和流水号回拨。

## 常见异常与排查顺序

1. 历史订单展示了新商品名/新价格：查是否错误使用了实时产品表字段而非快照字段。
2. 库存数对不上：先区分产品主记录聚合值、SKU 明细值和库存日志记录。
3. 预订单删了但正式出库单还在或反之：查两者联动逻辑和删除可见性同步。
4. 通知显示有事件但没外发：查 `NotificationEvent` 与 `NotificationDispatch` 是否同时写入。
5. 客服会话列表和消息详情不一致：查会话实体与消息实体的最新状态字段是否同步。

## 验证与回归关注点

- 改实体字段或关系后至少回归：列表展示、详情展示、导出、统计、删除治理、通知或库存联动。
- 改快照字段后优先回归：历史订单、预订单详情、报表和客服消息侧显示。

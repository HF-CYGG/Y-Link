# Y-Link 系统管理、用户中心与客服工作台

## 适用范围

- 适用于系统配置页、用户中心、客户端用户管理、客服工作台等管理端治理模块。
- 适用于判断“这个配置在哪个页、哪个后端路由负责、客服工作台和系统配置页怎么联动”的场景。

## 最后核对来源

- `src/views/system/*`
- `src/api/modules/system-config.ts`
- `src/api/modules/user.ts`
- `src/api/modules/client-user-manage.ts`
- `src/api/modules/notification.ts`
- `backend/src/routes/system-config.routes.ts`
- `backend/src/routes/user.routes.ts`
- `backend/src/routes/client-user-manage.routes.ts`
- `backend/src/routes/customer-service.routes.ts`

## 真实入口

- 系统管理总入口：`/system/*`
- 客服工作台入口：`/system/customer-service`
- 后端入口主要分为：用户路由、客户端用户路由、系统配置路由、客服工作台路由、通知路由。

## 前端链路

- 系统配置页拆成多个 section：流水号、O2O 规则、验证码通道、客服配置、部门树、通知配置等。
- 用户中心和客户端用户管理通过共享壳层做标签页切换。
- 客服工作台负责会话列表、详情、回复、指派、状态流转、内部备注和实时订阅。

## 后端链路

- `system-config.routes.ts` 承接：
  - 订单流水号
  - O2O 规则
  - 客服配置
  - 验证码平台
  - 客户端部门树
  - 客户职工目录导入导出与预览
- `customer-service.routes.ts` 承接：
  - 会话列表与详情
  - 回复
  - 指派
  - 状态变更
  - 结构化问题字段
  - 内部备注
  - 在线状态与 SSE

## 关键状态/字段/快照

- 系统配置不是单一 JSON，而是按 `configKey/configGroup` 维护多组配置。
- 客服工作台的关键会话字段包括：状态、优先级、负责人、内部备注、结构化问题字段、未读数。
- 客户端职工目录是独立治理对象，和客户端登录/部门树配置联动，但不等于普通用户表。

## 权限与安全边界

- 查看系统配置依赖 `system_configs:view`，修改依赖 `system_configs:update`，部分高风险项还要求 `admin` 角色。
- 客服工作台查看依赖 `customer_service:view`，回复/指派/改状态依赖 `customer_service:reply`。
- 职工目录导入、部门树修改、验证码通道修改都属于高风险治理操作，应视为后端配置变更而非纯前端表单改动。

## 常见异常与排查顺序

1. 系统配置页保存没生效：先查对应 `configKey` 写入逻辑，再查前端是否读到了旧缓存。
2. 用户中心标签切换错乱：查共享壳层 `viewKey` 和路由入口是否复用正确。
3. 客服会话列表和详情不一致：查会话详情接口是否触发已读/状态刷新逻辑。
4. 职工目录导入报错：查预览接口、部门树自动补齐逻辑和同名部门路径冲突。
5. 验证码/通知配置保存成功但通道不可用：继续看 `43` 的后端配置与外发通道文档。

## 验证与回归关注点

- 系统配置改动至少回归：读取、保存、刷新后回显、权限显隐。
- 客服工作台改动至少回归：列表、详情、回复、指派、状态切换、内部备注、在线状态。
- 涉及职工目录或部门树时回归：导入预览、正式导入、批量删除、路径冲突提示。

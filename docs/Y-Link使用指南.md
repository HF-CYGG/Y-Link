# Y-Link (EquipTrack) 使用指南

> 文档说明：本文面向最终使用人员与实施同事，介绍当前系统已经落地的核心能力、典型操作路径与使用注意事项。
> 适用范围：管理端（企业后台）与客户端（O2O 预订端）的日常使用培训、上线交接与运维答疑。
> 维护口径：本文只描述当前可交付能力；若后端安全策略、系统配置项或页面入口发生变化，必须同步修订本说明。

Y-Link 是一套面向文创与非遗行业的现代化出库管理系统，分为**管理端（企业后台）**与**客户端（O2O 线上预订端）**。本指南将帮助您快速了解系统的核心功能、业务流程及操作规范。

---

## 1. 核心业务闭环

Y-Link 致力于打通“线上预订 -> 线下核销 -> 出库追踪”的全链路：

1. **线上预订（客户端）**：用户在客户端（O2O大厅）浏览已上架的商品，加入购物车并提交预订订单。系统会实时冻结对应的商品库存。
2. **线下核销（管理端）**：用户凭订单信息到线下提货，管理人员在后台的“出库明细”中进行订单核销操作。
3. **出库追踪（管理端）**：核销完成后，系统正式扣减物理库存，生成出库流水，数据看板（Dashboard）的统计信息同步更新，完成业务闭环。

## 2. 管理端（企业后台）操作说明

### 2.1 系统初始化与登录
- **初始账号**：系统首次部署（无论是 Docker 还是本地）启动后，会自动初始化一个超级管理员账号。出于安全要求，管理员初始化密码必须通过后端私有配置 `INIT_ADMIN_PASSWORD` 提供，启动日志只会提示账号已初始化，不再输出明文密码。
- **密码修改**：为了系统安全，首次使用初始账号（默认 `admin`）登录后，请立即点击右上角个人头像，进入“个人中心”修改密码。

### 2.2 数据看板 (Dashboard)
数据看板提供直观的系统运营概览：
- **核心指标**：今日新增出库、待核销订单数、预订转化率等。
- **图表分析**：近 7 日出库趋势图（折线图）与商品类目/客户占比图表（饼图）。
- **排行与下钻**：提供“商品热销榜”与“活跃客户榜”，点击榜单项可展开抽屉，进行深度的下钻数据分析。

### 2.3 基础数据管理
- **标签管理**：维护商品的分类标签（如“文创周边”、“非遗手作”等），用于后续检索和统计。
- **商品管理**：
  - 录入商品的基础信息（名称、价格、规格）及库存数量。
  - **O2O上架**：在商品编辑中，可配置该商品是否在客户端的“商品大厅”展示。

### 2.4 出库管理
- **出库录入 (Order Entry)**：
  - **键盘流极速开单**：支持纯键盘操作快速检索商品、调整数量，系统实时计算总金额，极大提升业务员录入效率。
  - **草稿箱功能**：录入过程中支持保存为草稿，即便临时切换到其他页面，返回后仍可恢复输入内容。
- **出库明细 (Order List)**：
  - 查看所有历史出库单以及来自客户端的线上预订状态（待核销 / 已完成 / 已取消）。
  - 支持执行**订单核销**操作。
  - 提供出库凭证的 PDF 打印与导出功能。

### 2.5 系统设置与安全配置
- **系统配置**：
  - **O2O 预订规则**：可在此开启或关闭“单人限购”模式，并调整相关参数。
  - **验证码与通知配置**：可开启/关闭图片验证码、短信、邮件验证码功能；页面内提供“发送测试短信/邮件”按钮，方便与第三方平台联调。
- **用户与权限管理**：
  - **系统用户管理**：管理后台操作员账号，分配相应权限。
  - **客户端用户管理**：查看并管理在 O2O 端注册的用户，支持查看用户状态、修改资料，以及在二次确认下为其重置密码。
- **审计日志**：
  - 自动记录系统中所有关键的敏感操作和数据变更记录，方便安全追溯。

## 3. 客户端（O2O 预订端）操作说明

### 3.1 注册、登录与账号安全
- **多通道登录**：支持使用自定义用户名、手机号或邮箱任意一种方式进行登录，灵活便捷。
- **智能验证码降级**：
  - 登录验证码仅在输入一次密码错误后出现。
  - 系统优先使用后台配置的短信/邮件验证码；若未配置或发送失败，系统将自动降级回退至图片验证码，确保正常验证。
- **找回密码**：系统严格限制，仅在后台同时正确配置了手机短信和邮箱验证通道时，才允许自助找回密码；否则用户需联系管理员手动重置。

### 3.2 商品大厅与实时库存
- **商品大厅 (Mall)**：以现代化的网格/瀑布流布局展示所有已上架的商品。
- **库存透明**：每个商品卡片均会清晰展示“剩余可预订数量”（已扣除别人预订的数量）与“已被预订数量”，防止超卖。

### 3.3 购物车与预订结算
- 用户可将心仪商品加入购物车，并在“购物车”页面统一管理。
- 进入“结算页 (Checkout)”提交订单后，系统将锁定对应库存，并生成预订订单及核销凭证。

### 3.4 个人中心
- **单据追踪**：在个人中心可随时查看历史预订订单的状态（待核销/已完成）。
- **资料管理**：支持用户自助修改个人自定义用户名、绑定邮箱与手机号等基础信息。

# Y-Link 项目全局规范

## 0. Codex 工作总则

- 本文件是 Y-Link 仓库级开发规范。所有代码修改、重构、修复、测试、文档更新均必须遵守。
- Codex 在动手修改前，必须先判断当前任务是否命中仓库内 `.agents/skills/*/SKILL.md` 中定义的 skill。
- 若任务命中 skill，必须优先使用对应 skill 的工作流；若未使用，应在最终回复中说明未使用原因。
- 所有代码注释、UI 文本、计划书、提交说明、回复总结必须使用**简体中文**。
- 不允许为了快速修复绕过既有服务层、权限层、路由层、审计层或验证脚本。
- 修改核心业务代码后，必须给出验证命令和验证结果；无法运行时必须说明原因。

---

## 1. 强制性能验证（必须遵守）

- **触发时机**：每次大版本升级、核心页面重构、路由结构调整、依赖大版本更新、核心业务链路重构后，必须执行性能与回归验证。
- **验证方式**：在提交或结束当前大迭代前，必须运行：

```bash
npm run verify:performance

# 文件说明
- 本文档用于管理端上线前权限核对，确保角色授权、页面可见性、接口鉴权和审计留痕口径一致，避免上线后出现越权或误拦截问题。

# 管理端上线前权限核对清单

## 1. 适用范围
- 适用系统：Y-Link 管理端（企业后台）。
- 适用角色：管理员（admin）、操作员（operator）、供货方（supplier）及自定义角色。
- 执行阶段：联调完成后、灰度前、正式上线前各执行至少 1 次。

## 2. 角色基线确认
- [ ] 已确认当前权限基线文档为：`spec/admin-operator-permission-matrix.md`。
- [ ] 已确认管理员具备：`orders:delete`、`system_configs:update`、`users:*`、`audit_logs:*`。
- [ ] 已确认操作员不具备：系统配置修改、用户治理、审计导出等高风险权限。
- [ ] 已确认供货方仅开放供货业务所需接口，不开放管理端治理能力。
- [ ] 已确认所有自定义角色采用“最小权限”原则，不复制管理员全量权限。

## 3. 菜单与页面可见性核对
- [ ] 使用管理员账号登录，管理端高权限菜单均可见（系统配置、用户管理、审计日志、数据库迁移助手）。
- [ ] 使用操作员账号登录，仅展示其授权菜单，不出现高危治理入口。
- [ ] 使用供货方账号登录，管理端主业务菜单不可见或不可访问。
- [ ] 对每个角色逐一验证：直接输入页面 URL 时，未授权页面会被正确拦截。
- [ ] 路由守卫与页面按钮状态一致，不出现“菜单可见但核心按钮全部禁用”的误导。

## 4. 按钮与动作权限核对
- [ ] 管理员可执行出库删除、系统配置修改、用户状态变更、密码重置、审计导出。
- [ ] 操作员可执行日常开单、查单、核销、商品与标签维护，但无法执行高危治理动作。
- [ ] 供货方仅可执行供货流程中被授权的动作，不可触发管理端治理按钮。
- [ ] 所有禁用态按钮有明确文案提示（如“当前角色无权限”）。
- [ ] 不存在前端按钮放开但后端固定 403 的长期配置错误。

## 5. 接口鉴权一致性核对
- [ ] 管理员测试集：关键治理接口返回 200，业务结果正确。
- [ ] 操作员测试集：日常业务接口返回 200，高危接口返回 403。
- [ ] 供货方测试集：仅供货相关接口可用，其余接口返回 403。
- [ ] 菜单、按钮、接口三层口径一致（无“前端放行/后端拦截”或“前端拦截/后端放行”）。
- [ ] 对关键接口执行回归：`/api/system-configs/*`、`/api/users/*`、`/api/audit-logs/*`、`/api/data-maintenance/*`。

## 6. 审计与追踪核对
- [ ] 已验证系统配置修改会写入审计日志，记录操作者与变更内容。
- [ ] 已验证数据库迁移相关动作会写入审计日志，记录动作类型与结果。
- [ ] 已验证用户重置密码、启停用等敏感动作会写入审计日志。
- [ ] 已验证审计日志支持按时间、操作者、动作类型检索。
- [ ] 已验证异常越权请求（403）具备可追踪的安全日志线索。

## 7. 上线放行结论
- [ ] 权限配置核对通过：通过 / 不通过（圈选其一）。
- [ ] 风险项已记录并给出责任人与修复截止时间。
- [ ] 产品、开发、测试、运维四方已完成上线前权限会签。

# 全量权限矩阵与门禁说明

> 文档更新时间：2026-05-01
> 说明：本文档作为当前权限治理真源，旧版权限表若与本文冲突，一律以本文为准。

## 1. 角色与权限基础

### 1.1 角色
- `admin`：系统治理与高危数据操作负责人。
- `operator`：日常业务处理人员。
- `supplier`：供货方账号，仅处理送货与历史查询。

### 1.2 权限点总数
- 当前共 `26` 个权限点。

## 3. 默认角色权限摘要
- `admin`
  - 默认拥有除 `inbound:create` 之外的全部治理与业务权限。
- `operator`
  - 仅保留日常业务操作，不进入系统治理、用户治理与审计域。
- `supplier`
  - 仅保留供货相关入口：查看商品、创建送货单、查看自身历史。

## 4. 门禁落点说明

### 4.1 前端门禁
- 路由入口：
  - 由 `src/router/routes.ts` 的 `requiredPermissions / requiredAnyPermissions / allowedRoles` 决定。
- 页面动作：
  - 统一使用 `usePermissionAction` 做按钮显隐与点击前兜底。
- 导航可见性：
  - 只展示当前角色至少有一个可访问子路由的菜单。

### 4.2 后端门禁
- 接口守卫：
  - 通过 `requirePermission(...)` 控制动作能力。
- 高危治理接口：
  - 需要时叠加 `requireRole('admin')`，形成权限点 + 角色双门禁。

## 6. 高危动作审计要求
- 以下动作必须有审计记录：
  - 系统配置更新
  - 验证码平台测试
  - 数据备份与导入
  - 数据库迁移预检、执行、切换、回退、清理覆盖
  - 用户创建、编辑、启停、重置密码
  - 关键业务状态修改

## 7. 回归检查清单
- `admin / operator / supplier` 三个角色都要验证。
- 验证顺序：
  - 菜单可见性
  - 页面按钮可用性
  - 接口返回码
  - 审计是否留痕
- 重点核对：
  - 操作员不能进入系统治理与用户治理域。
  - 供货方不能进入普通管理端工作台。
  - 高危动作不能出现“页面隐藏但接口可调通”或“页面可点但接口 403”。
